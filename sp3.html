Overview

Timeline: 01/2016
Project: Personal
Type: Web App Redesign

SP3 is a seismic performance prediction tool that enables engineers to quickly obtain a comprehensive prediction of building damage through our carefully designed and user-friendly application.

Original Product Overview:

Problems Found

1. Too many jargons on the SP3 tool. It is very hard for laypeople to understand all the terminologies and use the tool.
2. The process is too lengthy and provides too much details/information. It also appears that in the tutorial most fields used default value to run the analysis. However, these fields are still shown to the client.
3. The user flow is not very intuitive nor friendly.


Solutions

• I created a standalone version of the software in an “easy mode”. While all the internal process is the same, the easy mode only presents minimal necessary information and uses most common values for each field based on the information given by the user.

    a. Example: If the building address is 123 Main Street, AA, 48105, the building age is around 50 years. The software will look in a database to find the most common values for the fields based on all the other buildings with similar statistics, e.g. all the buildings within a 10-mile radius to 123 Main Street with building age between 45-55 years. Then it will fill the fields with these values.
    b. For more specific data that cannot be generated through step a., the software will prompt the users to enter these data. However, there will be an option to help the user figure out the value by asking the user some specific questions that are easy to answer, and speculate the value based on these answers. (e.g. does your building shake when you stomp on the floor? Yes -> made of wood. No -> made of concrete)

• The dashboard gives a summary overview of the analysis by expanding upon clicking. This saves the user’s time and presents them with all the essential information from the analysis.

• The user does not get redirected back to the dashboard after creating the project as shown in the tutorial. The user is guided through the whole process without being automatically redirected to the dashboard.

• The analysis option is moved to be the last step. This makes the process more intuitive.

• Every terminology that might be confusing to the user has an information icon. By hovering over the icon, the user can see the definition of the terminology explained in a way that any layperson could understand.

• The result is presented with an interactive graph. The user is able to get specific values from the graph by hovering their mouse and move it along the graph. This way the user can not only get a visual representation of the graph, but is also to have accurate data for exact points on the graph.
